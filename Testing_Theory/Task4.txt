1. Что такое Тест дизайн?
    Тест дизайн - процесс, который позволяет используя определенные техники, создать оптимальное тестовое покрытие тестируемого объекта с 
    минимально затраченным временем. Главная задача тест-дизайна, это разработать тестовые сценарии для проверки максимального колличества 
    функций за минимальное время.
    Проще говоря, цель тест-дизайна - создать максимально эффективный набор кейсов, покрывающий наиболее важные аспекты тестируемого ПО, т.е. 
    минимизировать количество тестов, необходимых для нахождения большинства серьезных ошибок


2. Для чего нужны Техники тест дизайна?
    - чтобы разработать тесты, которые обнаружат наиболее серьезные ошибки
    - чтобы минимизировать количество необходимых тестов для проверки продукта
    

3. Что такое Классы эквивалентности?
    Набор данных, которые подаются на ввод в модуль и приводит к одинаковому результату. Методика группировки и разделения тестовых входных
    данных на эквивалентные классы.
    Правила:
        - Определение класса эквивалентности
        - Хотя бы 1 тест для одного класса
    Возраст, тестирование формочек, ценовые скидки, где есть какое-то количество, временные зоны, перевозки груза - вес, объемы, расстояние, 
    способы доставки, загрузка файлов в нашу систему(размер, форма, разрешение, название), зависимость цены от времени дня, фильтры на сайте
    интернет магазина

    
4. Что такое Граничные значения?
    Граничные значения — это значения на границе допустимого диапазона входных данных, которые могут привести к изменению поведения программы.
    Это те места, в которых один класс эквивалентности переходит в другой


5. Что такое Таблица принятия решений?
    Это техника тестирования, которая помогает нам визуально отобразить то, как будет вести себя программа при заданных условиях. Когда надо в 
    зависимости от выбранных параметров, сказать программе как себя вести.
    Например при заполнении формы при заданных параметрах должны происходить те или иные действия
    2^3              1 2 3 4 5 6 7 8
    Баланс           0 0 0 0 1 1 1 1
    Овердрафт        0 0 1 1 0 0 1 1
    Овердрафт +/-    0 1 0 1 0 1 0 1
    Решение          0 0 0 1 1 1 1 1

    

6. Что такое Попарное тестирование?
    Этот метод основывается на тестировании всех возможных пар значений параметров. Предполагается, что большинство ошибок вызывается 
    взаимодействием не более двух параметров. Парное тестирование значительно уменьшает количество необходимых тестов по сравнению с полным 
    комбинаторным тестированием.


7. Перечислите Методы комбинаторного тестирования
    1. Полное комбинаторное тестирование - проверяются все возможные комбинации значений входных параметров
    2. Парное тестирование(Pairwise testing)
    3. N-wise тестирование - Обобщение парного тестирования, где n может быть больше двух. Например, если n=3, то тестируются все возможные 
    тройки значений параметров
    4. Комбинированное тестирование - Метод, при котором некоторые комбинации параметров тестируются полностью, а другие – только частично, в 
    зависимости от вероятности возникновения ошибок или критичности тестируемых сценариев
